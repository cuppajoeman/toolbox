Mixamo batch animation import workflow

This workflow is for a Blender character that already has its real mesh and main armature, and a folder of Mixamo animation FBX files that contain armatures with animation data. The goal is to keep one character file and turn each Mixamo FBX into a reusable Blender Action on the existing armature.

The important assumption is that the character armature is in a T-pose rest pose before Mixamo animation data is applied. Mixamo animation downloads are authored against that T-pose assumption. If the character is in an A-pose rest pose, legs may still look acceptable, but arms often come in wrong because the local arm rotations are interpreted from the wrong rest orientation.

Files

  • Add-on: mixamo_batch_action_importer.py
  • Install location example: %APPDATA%\Blender Foundation\Blender\5.1\scripts\addons\mixamo_batch_action_importer.py
  • Panel location: 3D View > Sidebar > Animation > Mixamo Batch

Expected workflow

  1. Start with the character in Blender: mesh, armature, and weight painting already set up.
  2. If the character is not in a T-pose rest pose, convert it first. See Convert an A-pose character to T-pose.
  3. Export the character to FBX from Blender.
  4. Upload that FBX to Mixamo.
  5. Download the animations or animation pack from Mixamo using the default settings.
  6. Open the original character file in Blender.
  7. Select or choose the main armature in the add-on panel.
  8. Choose the folder containing the Mixamo animation FBX files.
  9. Run Import Mixamo Actions.

Recommended add-on settings

  • Transfer Mode: Direct Copy Curves
  • Disable IK In Imported Actions: enabled
  • Rename From Filename: enabled
  • Mark Fake User: enabled
  • Delete Temporary Imports: enabled
  • Strip Mixamo Prefix: enabled
  • Ignore *_end Bones: enabled
  • Overwrite Existing Actions: enabled when re-importing the same animation set

What the add-on does

  • Imports each FBX in the selected folder.
  • Finds the temporary imported armature and its Action.
  • Creates a clean Action on the real selected armature.
  • Copies matching F-curves by bone name.
  • Ignores harmless terminal bones such as *_end.
  • Keys IK constraint influence to 0 inside the imported Action when Disable IK In Imported Actions is enabled.
  • Deletes the temporary imported armature and temporary source Action.
  • Keeps the final scene as one character armature plus many reusable Actions.

Why T-pose matters

The add-on transfers animation primarily by bone name and local transform curves. That is simple and reliable when the source Mixamo armature and the target character armature have the same rest pose. If the target character is in A-pose, the arm bones start from a different local orientation, so the same animation curves rotate the arms from the wrong basis.

In practice, this can look like the legs working while the arms are twisted, lowered, or otherwise incorrect. The legs are often close enough between rigs, but arms can differ by 50 degrees or more between A-pose and T-pose.

Troubleshooting

  • If the Action appears bound to Armature.001, the file probably contains leftover temporary Actions from an old import. Re-run from a clean character file with the current add-on.
  • If the character is stiff, make sure Disable IK In Imported Actions is enabled. IK constraints can override imported FK animation curves.
  • If legs work but arms are wrong, check whether the character's rest pose is A-pose instead of T-pose.
  • If an FBX imports with no Action, it may be a reference/startup FBX rather than an animation. The add-on can skip it.
